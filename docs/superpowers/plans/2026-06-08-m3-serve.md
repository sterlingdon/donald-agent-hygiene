# donald-agent-hygiene M3 Implementation Plan — `serve` (local one-click execute)

> Builds on M1/M2. REQUIRED SUB-SKILL: executing-plans / subagent-driven-development. Steps use `- [ ]`.

**Goal:** Add a `serve` subcommand that hosts the hygiene report on a local web server with per-item **Execute** buttons that run the cleanup (with backup) — realizing the original "HTML 报告内起本地服务一键执行" decision.

**Architecture:** New `hygiene/serve.py`: pure helpers (`actionable_index`, `apply_one`, `render_interactive_html`) + a `BaseHTTPRequestHandler` factory + `build_server()`. `cli serve` wires it. Reuses M2's `actions.execute` for the actual mutation (dry_run=False, always backs up first).

**Security invariants (non-negotiable):**
- Bind **127.0.0.1 only** (never 0.0.0.0).
- A per-process random **token** gates the mutating POST endpoint.
- Browser **confirm()** before any execute.
- `actions.execute` already backs up before mutating; Codex MCP stays command-only.

**Tech Stack:** stdlib `http.server`, `urllib`, `secrets`, `json`, `threading` (tests). pytest from `scripts/`.

---

## Task S1: `serve.py` core

**Files:** Create `scripts/hygiene/serve.py`, `scripts/tests/test_serve.py`

Helpers:
- `actionable_index(findings) -> list[(idx, finding)]`: green/yellow findings whose `item.kind` in {skill, mcp} (the ones `actions.execute` can act on).
- `apply_one(findings, idx, backups, home, dry_run=False) -> ActionResult`: `actions.execute(findings[idx], ...)`.
- `render_interactive_html(findings, token, meta) -> str`: M2 `report.render_html` + an appended "一键处置" panel with one `<button onclick="ah(i)">execute</button>` per actionable index and a small JS `ah()` that `confirm()`s then `fetch('/apply?i=..&t=<token>', {method:'POST'})` and writes the JSON result back.
- `make_handler(findings, token, backups, home)`: `BaseHTTPRequestHandler` subclass — `GET /` → interactive HTML (200); `POST /apply?i&t` → token check (403 if wrong), index bounds (400), else `apply_one(..., dry_run=False)` and return JSON `{applied,kind,command,backup}`; silence logging.
- `build_server(findings, backups, home, port=0) -> (httpd, token)`: `HTTPServer(("127.0.0.1", port), handler)` with a `secrets.token_urlsafe(8)` token.

Tests: actionable_index filters correctly; render contains an execute button + the token + a `/apply` fetch; apply_one archives a temp skill (backup made); a live-server test (thread + urllib) asserts `GET /` → 200 and `POST /apply` without token → 403, with bad token → 403.

## Task S2: `cli serve` subcommand

**Files:** Modify `scripts/hygiene/cli.py`; add `scripts/tests/test_cli_serve.py`

Add `serve` subparser (same path args + `--backups`, `--port` default 8765). Handler builds findings via `_build_findings`, calls `serve.build_server(...)`, prints `http://127.0.0.1:<port>/` + token, `serve_forever()` with KeyboardInterrupt handling. Test: a `--port 0` variant returns the server object without blocking (factor the blocking `serve_forever` out so it's testable) — assert `build_server` address is `127.0.0.1`.

## Task S3: real local smoke + SKILL.md

Boot `build_server` on port 0 in-process against real data, GET `/`, assert 200 + contains "处置", confirm bound to 127.0.0.1, shut down. Update SKILL.md usage with the `serve` command. Confirm no mutation happened (read-only GET).
